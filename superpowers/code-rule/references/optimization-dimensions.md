# 优化维度

## 代码可维护性维度

### 基础规范

遵守前端开发的基础规范，确保代码质量、可维护性和团队协作。
使用 ESLint、Prettier 和代码审查来确保规范的遵守。

### 高内聚，低耦合

**高内聚：**
- 组件内：组件包含与其功能相关的所有代码和资源，紧密相关
- 功能内：将特定功能的代码组织在独立模块中

**低耦合：**
- 组件间：通过 Props/Events 通信，不直接访问内部状态
- 模块间：减少依赖关系

**实现方法：**
1. 组件化开发：拆分为小可重用组件，每个处理单一功能
2. 单一职责原则：每个组件/模块只负责一个功能
3. 明确定义接口：使用属性、事件或函数通信
4. 依赖注入：减少硬依赖，便于替换和测试
5. 模块化开发：减少全局变量和依赖

### 基础性工作完善程度

1. **前端框架统一** — Vue、React、ReactNative、Flutter、UniApp
2. **UI 库统一** — ElementUI、AntDesign、AntDesignMobile 等
3. **构建工具统一** — Webpack、Vite、VueCLI、UmiJS、NextJS
4. **业务组件封装** — 沉淀通用业务组件库
5. **技术文档沉淀** — 每个工程师应养成写文档习惯
6. **性能优化** — 减少 HTTP 请求、CDN、图片优化、懒加载、字体图标、合并压缩、浏览器缓存、异步加载、DOM 操作、SSR、性能检测、代码分割等（参考雅虎前端优化 35 条规则）
7. **学习精神** — 持续关注新技术和最佳实践
